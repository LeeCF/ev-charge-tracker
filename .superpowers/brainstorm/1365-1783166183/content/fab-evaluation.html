FAB 可行性评估 + 优化方案

悬浮加号按钮（FAB）是成熟设计模式。关键在于解决3个常见问题。

原始 FAB 的 3 个问题
遮挡内容：右下角固定悬浮会遮挡记录列表最后一行，用户以为没有更多内容
功能不明确：裸"+"图标没有文字，新用户不知道点击后会发生什么
位置与 Tab 栏重叠：右下角 FAB 和 Tab 栏同区域，视觉竞争，拇指误触率高

C · FAB 长按展开快捷选项

轻触打开完整表单；长按弹出「慢充满充 / 快充满充 / 自定义」三个快捷项，最常见操作一步完成。像 iOS 长按 App 图标的快捷菜单。

FAB 尺寸规范（符合移动端标准）
圆形 FAB：56×56dp（标准），最小 48×48dp，圆角 16dp（圆角方形）或 50%（圆形）
扩展 FAB：高度 56dp，宽度自适应内容，左右内边距 20dp，图标与文字间距 12dp
位置：距屏幕右边 16dp，距 Tab 栏上方 16dp（悬浮，不重叠 Tab）
内容底部 padding：滚动区需加 padding-bottom = FAB 高度(56) + 与 Tab 距离(16) + Tab 高度(49) = 约 120dp，防止最后一行被遮挡
阴影：0 4px 20px rgba(0,102,255,0.42)，与英雄卡蓝色光晕呼应
